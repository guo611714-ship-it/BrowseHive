# Hermes + Obsidian + LLM Wiki 构建 AI 知识库

## 概述

这套工作流用 **Hermes**、**LLM Wiki** 和 **Obsidian** 三个工具串联，实现完全自动化的个人知识管理：

- **文档导入** → AI 自动整理 → **Wiki 生成** → 双向链接浏览
- 全程无需手动操作图形界面
- 所有繁琐的知识管理工作交给 AI，只需提问和探索

---

## 第一部分：Obsidian 介绍

### 什么是 Obsidian

Obsidian 是一款**本地双向链接笔记工具**，完全免费，跨 Windows/macOS/Linux 三大平台。

### 为什么选择 Obsidian 而非 Notion/飞书

| 特性 | Obsidian | Notion/飞书 |
|------|----------|-------------|
| 数据存储 | 完全本地，Markdown 文件 | 云端，数据属于平台 |
| 双向链接 | 原生支持，核心功能 | 支持但非核心 |
| 隐私 | 数据永远属于自己 | 需信任服务商 |
| 费用 | 免费，个人使用无限制 | 付费版功能更多 |

### 核心功能：双向链接

**双向链接**能把零散的笔记织成一张网。

**示例：**
- 笔记 A：`参考苹果的种植方法`
- 笔记 B：`苹果是一种水果`

这两篇笔记通过"苹果"这个词条**自动关联**，点击任意一个都能跳转到另一个。

**传统笔记软件的问题：**写了几十篇笔记，笔记之间相互孤立，根本不知道"苹果"这个词在其他所有地方出现过。

### Obsidian 的安装与配置

1. 打开官网 [obsidian.md](https://obsidian.md) 下载
2. macOS 版本安装后打开 APP
3. 选择或创建一个 **Vault**（保险库）

> **Vault 是什么？** 本质上就是一个文件夹，里面装满了 Markdown 文件。可以把已有的笔记文件夹直接导入，也可以新建一个空白的开始写。

### 双链的用法

在笔记里输入**两个方括号** `[[`：

```
[[水果]]
```

Obsidian 会自动把它变成一个**蓝色链接**：
- 如果「水果」这篇笔记存在 → 点击即可跳转
- 如果不存在 → 点击会创建它

### Graph View（知识图谱）

按 `Cmd + \` 打开 Graph View，Obsidian 会把所有笔记及链接关系画成一张图：

- 一眼看清知识结构
- 识别哪些节点是孤岛（孤立笔记）
- 识别哪些是枢纽（核心知识节点）

## 第二部分：LLM Wiki 介绍

### LLM Wiki 是什么

GitHub: `nasui/LLM_wiki`（3500+ stars），采用 GPL V3 协议

**核心特点：持久化的知识库**，与传统的 RAG 系统完全不同。

### 传统 RAG vs LLM Wiki

| 维度 | 传统 RAG | LLM Wiki |
|------|----------|----------|
| 工作方式 | 每次提问从零搜索文档 | AI 增量构建持久化 Wiki |
| 知识积累 | 无，每次重新拼凑答案 | Wiki 不断积累和更新 |
| 知识关联 | 无 | 页面之间形成引用和关联 |
| 矛盾处理 | 无 | 矛盾处会被标注 |
| 知识空白 | 不提示 | 会被识别出来 |
| 回答来源 | 不一定标注 | 标注来源，告知来自哪篇文档 |

### LLM Wiki 的安装步骤

1. 访问 `github.com/nasui/LLM_wiki/releases`
2. 下载 macOS 的 `.dmg` 或 `.tar.gz` 文件
3. 解压后将 `LLMwiki.app` 拖到「应用程序」文件夹
4. 双击打开 APP
5. 首次使用点击 **New** 或 **+** 按钮新建项目
6. 点击右上角**设置图标**，填入 API

> **支持的 API：** OpenAI、Claude、MiniMax 以及任何 OpenAI 兼容的 API 接口

### 文档导入方法

- 左侧文件区右键点击 → 选择 **Import**
- 或按 `Cmd + I` 快捷键

**支持格式：** PDF、Markdown、TXT

### 界面结构

- **左侧文件区**：展示已导入的文档
- **底部对话框**：直接提问，AI 结合文档内容回答
- **右侧预览区**：实时渲染成结构化的 Wiki 页面
- **Wiki 文件夹**：`项目目录/wiki/` 里装的是 AI 生成的 Wiki 页面（纯 Markdown）

> 这个 Wiki 文件夹可以直接拖进 Obsidian 作为 Vault 使用，Obsidian 的双向链接功能**全部兼容**。

## 第三部分：Hermes 的角色

### Hermes 在工作流中的定位

**自动化执行引擎** — 通过 AppleScript 控制 LLM Wiki 的图形界面。

**含义：** 不需要手动去打开 LLM Wiki、找到导入按钮、选择文件。只需告诉 Hermes：

> "把桌面的产品文档导入知识库"

Hermes 就会自动完成全部操作。

### 必要配置：开启辅助访问权限

这是 macOS 的安全机制，开放后应用程序才能控制其他应用的界面。

**操作步骤：**
1. 打开「系统设置」→「隐私与安全性」→「辅助功能」
2. 把 Terminal 或你的终端应用添加到列表里
3. 开启开关

### Hermes 能自动化执行的操作

- 自动完成 LLM Wiki 的文档导入
- 自动在对话框里输入问题并发送
- 自动读取右侧 Wiki 预览区的内容

### 重要规则：只有明确要求才会操作知识库

| 你说的话 | Hermes 行为 |
|----------|-------------|
| "写入知识库"、"导入知识库"、"把这个文件放进知识库" | 执行导入操作 |
| "结合知识库"、"查一下知识库"、"根据知识库回答" | 检索知识库 |
| 日常普通对话（如"今天天气怎么样"） | **不主动读取**知识库 |

**好处：** 知识库不会被无关对话污染，每次检索结果更精准。

## 第四部分：三工具串联完整流程

### 工作流架构图

```
用户（下达指令）
    ↓
Hermes（接收指令，AppleScript 自动化）
    ↓
LLM Wiki（AI 读取文档，提取实体，增量更新 Wiki）
    ↓
Wiki 文件夹（项目目录/wiki/）
    ↓
Obsidian Vault（双向链接，知识图谱浏览）
    ↓
用户（提问探索）
```

### 完整链路演示

#### 第一步：下达指令

> "把桌面的产品需求文档导入知识库"

→ 包含明确动作「导入」，Hermes 知道需要执行知识库写入操作

#### 第二步：Hermes 执行自动化

通过 AppleScript 自动完成：
1. 激活 LLM Wiki 窗口
2. 点击 Import 按钮
3. 在文件选择器里定位到桌面
4. 选中目标文件
5. 确认导入

**全程不需要手动操作**

#### 第三步：LLM Wiki 处理文档

AI 开始：
1. 读取文档内容
2. 识别关键实体（产品名称、功能模块、技术方案）
3. 在 Wiki 里创建或更新对应页面
4. 右侧预览区实时显示生成的内容：
   - 摘要
   - 核心要点
   - 来源标注

#### 第四步：展示知识网络

1. 打开 LLM Wiki 项目目录里的 `wiki` 文件夹
2. 作为 Vault 拖进 Obsidian

→ 现在你拥有了一个由 AI 精心整理过的知识库

### 循环迭代

整个流程可以**无限循环**：

- 导入更多文档 → Wiki 不断扩充
- 已有页面被补充新信息
- 与新内容矛盾处被标注
- 不同文档的同一概念被关联到同一 Wiki 节点

→ 知识库越来越准确、越来越丰富

## 第五部分：界面功能详解

### LLM Wiki 界面

**左侧文件区上下文菜单：**
- 右键点击文件区 → 弹出菜单 → 选择 **In**（导入入口）
- 或按 `Cmd + I` 打开导入对话框

**对话与 Wiki 生成示例：**
- 提问：`hermes cloud code 怎么配合使用`
- AI 给出完整回答
- 右侧预览区渲染结构化 Wiki 页面，包含：
  - 摘要
  - 核心要点
  - 工具对比表格
  - 关键概念解释
  - 关联的工具词条

**回答溯源：**
- 每个回答会标注来源
- 标注具体参考了哪篇文档的第几段

### Obsidian 界面

**双链笔记示例：**
笔记内蓝色高亮词条：`hermes`、`cloud code`、`LLM wiki` 等 → 点击跳转到对应笔记页面

**Graph View：**
- 按 `Cmd + G` 打开
- 每个圆点代表一篇笔记
- 连线代表双向链接关系
- 中心节点 = 核心知识枢纽
- 边缘孤立节点 = 需要补充关联的知识点

**Obsidian 与 LLM Wiki 的兼容性：**
- LLM Wiki 的 `wiki` 文件夹不需要任何转换，直接拖进 Obsidian
- 所有 Wiki 页面 Obsidian 都能完美识别
- 所有 Wiki Link 都可以点击跳转

## 总结：三条核心规则

### 规则一：说"写入知识库"，Hermes 来导入

当你有新文档（产品文档、会议记录、学习笔记、技术方案）需要管理时：

> 告诉 Hermes "把桌面的某某文件导入知识库"

Hermes 自动完成从打开 LLM Wiki 到确认导入的全部操作。

**前提：** macOS 已开启辅助访问权限（第一次配置需要 1-2 分钟，之后无需再管）

### 规则二：说"结合知识库"，Hermes 来检索

当你需要基于资料库回答问题时：

> "根据我们已有的产品文档，介绍一下我们的技术架构"

加上"结合知识库"这几个字，Hermes 去 LLM Wiki 检索相关文档，AI 整合后给出完整回答并标注来源。

**日常对话**（如问天气）Hermes **不会**去查资料库，保证知识库不被污染。

### 规则三：Obsidian 随时可用

LLM Wiki 的 `wiki` 文件夹：
- 不需要任何转换
- 直接拖进 Obsidian 作为 Vault 使用
- 双链跳转、Graph View、全文搜索 — 所有 Obsidian 功能随时可用

可以在 Obsidian里边浏览边做笔记，Obsidian 内容是纯 Markdown，可以在任何工具之间迁移。
